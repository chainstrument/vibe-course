# Product Requirements Document - Vibe Course

## Executive Summary
Vibe Course is an interactive learning platform designed to teach developers **vibecoding** - the methodology of collaborative programming with AI. Since users bring their own AI keys, the platform focuses on education, guidance, and showcasing best practices rather than direct AI integration.

## Vision
Empower developers to become proficient at leveraging AI in their coding workflow by teaching them how to effectively communicate, iterate, and build software collaboratively with AI assistants.

## Market & Audience
- **Primary**: Junior to mid-level developers wanting to learn AI-assisted development
- **Secondary**: Educators and coding bootcamps
- **Tertiary**: Developers curious about AI collaboration models

## Core Features (MVP)

### 1. Guided Projects
- **Description**: Structured, step-by-step projects that teach vibecoding methodology
- **User Flow**: 
  - Select a project (e.g., "Build a Todo App with Vibecoding")
  - Follow guided steps showing how to prompt the AI effectively
  - See example prompts and expected approaches
  - Users execute prompts with their own AI
  - Self-validate progress by reviewing transcripts
- **Example Projects**: 
  - Build a CLI tool
  - Create a small web app
  - Debug & optimize existing code

### 2. Challenges
- **Description**: Exercises where users apply vibecoding skills
- **Levels**: Beginner, Intermediate, Advanced
- **Format**:
  - Problem statement
  - Suggested vibecoding approach
  - Template prompts
  - Expected transcript characteristics
  - Example solution transcript
- **User Flow**:
  - Attempt challenge with their own AI
  - Compare approach against best practices
  - Learn from example solutions

### 3. Learning Modules
- **Description**: Educational content on vibecoding fundamentals
- **Topics**:
  - What is vibecoding?
  - How to prompt effectively
  - Iterating with AI
  - Debugging in conversation
  - Anti-patterns and common mistakes
  - Project management with AI
- **Format**: Markdown/interactive pages with examples

### 4. Transcript Showcase
- **Description**: Gallery of vibecoding transcripts (good & bad examples)
- **Features**:
  - Annotated transcripts showing key moments
  - Analysis of effective vs ineffective prompts
  - Real examples of vibecoding workflows
  - Community-submitted transcripts (future)
- **Purpose**: Learning by example

### 5. Prompt Templates
- **Description**: Ready-to-use prompt templates for common scenarios
- **Templates**:
  - Initial feature request prompts
  - Debugging conversation starters
  - Refactoring requests
  - Code review & optimization
- **Customization**: Users can adapt templates to their needs

## Non-Features (Out of Scope for MVP)
- Direct AI integration / API calls
- User accounts & progress tracking (initially)
- Community forum
- AI model comparison tools
- Automated code execution/testing

## Success Criteria
1. Users can articulate what vibecoding is after using the platform
2. Users can execute a guided project successfully
3. Users can identify good vs bad vibecoding practices
4. Positive feedback on learning experience
5. Measurable improvement in user prompt quality (self-reported or transcript analysis)

## Roadmap

### Phase 1: MVP
- Basic project & challenge structure
- 2-3 guided projects
- 5-10 challenges (mixed levels)
- 3-4 learning modules
- Basic transcript examples
- Prompt templates library

### Phase 2: Enhancement
- User authentication & progress tracking
- User-submitted transcripts
- Advanced filtering/search
- Difficulty progression
- Analytics on user engagement

### Phase 3: Community
- Transcript sharing & voting
- Peer reviews
- Leaderboards / badges
- Integration with popular AI platforms
